Chapter 0: Introduction

0.1 Specification Terminology

Terminal symbols are shown in  bold, fixed-width  font. The definition of a terminal symbol appears in the productions of the lexical and syntactic grammars, and whenever the text directly refers to such a symbol in this specification. A terminal symbol must appear in a program exactly as written.

Nonterminal symbols are shown in  italic, bold, fixed-width  font. The definition of a nonterminal is introduced by the name of the nonterminal followed by a colon. One or more alternative right-hand sides for the nonterminal then follow on succeeding lines.

For example, the syntactic definition:

  LoopStatement:
    loop Statement

states that the nonterminal LoopStatement represents the token loop, followed by a Statement.

Another example:

  ArgumentList:
    Argument
    ArgumentList , Argument

This shows that an ArgumentList may represent either a single Argument or an ArgumentList followed by a comma and an Argument. This definition is recursive, meaning an ArgumentList may contain any positive number of arguments.

The informal grammar in this specification is bottom-up and left-recursive. A complete top-down, right-recursive grammar with disambiguation rules is included in an appendix.

The subscripted suffix opt, which may appear after a terminal or nonterminal, indicates an optional symbol. For example:

  ReturnStatement:
    return Expressionopt

is an abbreviation for:

  ReturnStatement:
    return
    return Expression

When the underlined word "either" follows the colon in a grammar definition, it signifies that each of the terminal symbols or tokens on the following lines is an alternative definition. For example:

  OctalDigit: either
    0 1 2 3 4 5 6 7

which is equivalent to writing each number on its own line.

Lexical definitions may exclude certain expansions using the underlined word "except" for example:

  Identifier:
    IdentifierName, except ReservedKeyword, Digit

Some nonterminals are defined descriptively when listing all alternatives is impractical. For example:

  RawInputCharacter:
     any ASCII character
